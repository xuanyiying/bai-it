# OpenEn — Claude 工作手册

> **新 session 第一件事：先看 [HANDOFF.md](./HANDOFF.md) 了解当前状态。**

## 项目简介

OpenEn 是一个纯本地的 Chrome 扩展，帮助用户在浏览英文网页时进行句子分块（chunking）和词汇学习。用户装完插件、填一个 LLM API key，就能用。零后端、零登录。

## 开发流程

每个功能模块从想法到上线，走四步：

1. **需求对齐** — 用户说想法，Claude 主动找漏洞、问清模糊点、补充没想到的场景。双方都理解并同意要做什么。
2. **验收标准** — 需求确认后、技术方案之前，定义可检查的完成条件。条件要具体、可验证。能自动化测试的写测试用例，不能的标注"需要用户确认"。
3. **技术方案** — Claude 出方案，技术约束翻译成大白话。有风险的点先做最小验证。如果技术约束影响产品方案 → 回到第一步。
4. **动手开发** — 决策实时更新文档。密钥只存环境变量。每个 session 结束更新 HANDOFF.md。

**核心原则：** 需求和技术之间是往返循环，不是线性流水线。不确定的事先确认再说结论。

## 开发规矩

- **密钥安全**：代码和文档中不出现任何明文 key，密钥只存环境变量或用户本地配置
- **任务追踪**：用 GitHub Issues，不设单独的 workstreams 文件
- **并行开发**：按文件拆分 worktree，不按功能拆（避免多 worktree 改同一文件导致冲突）
- **每个文档有明确受众和职责**，不重叠
